# ComfyUI PNG to WebP Migration Tool

A command-line script to convert a folder of PNG images generated by ComfyUI into optimized WebP images (lossy, 85% quality by default) while fully preserving and embedding their ComfyUI visual workflow and prompt metadata into EXIF tags.

This utility is adapted from the [ComfyUI-SaveCompressed-Weppy](https://github.com/skkut/ComfyUI-SaveCompressed-Weppy) repository.

## Features

- **WebP Compression**: Saves storage space by converting PNGs to compressed WebPs at customizable quality settings (default: 85% quality, lossy).
- **Metadata Preservation**: Extracts ComfyUI visual graph workflows and API prompts from PNG text chunks and embeds them into standard WebP EXIF tags (`Make` and `ImageDescription`), enabling ComfyUI to read workflows from the resulting WebP files via drag-and-drop.
- **EXIF Size Protection**: Automatically filters out heavy base64 data URIs (like embedded input images or masks) from the workflow JSON to prevent EXIF chunk length overflows.
- **Dry-run Mode**: Preview conversion operations, file count, and estimated space savings without writing any files or modifying directories.
- **Safety Safeguards**: Default settings prevent files from being overwritten or deleted.
- **Subdirectory Traversal**: Option to process folders recursively.
- **Multiprocessing Support**: Accelerates image conversion using Python's `ProcessPoolExecutor`, scaling performance with multi-core CPUs.

## Requirements

Ensure you have Python 3 installed. The script depends on the following libraries:

- `Pillow` (PIL) - For image loading and saving
- `piexif` - For writing EXIF headers inside WebP (falls back to native Pillow EXIF writing if not installed)

Set up the virtual environment and install dependencies:

```bash
# Create the virtual environment
python -m venv venv

# Activate the virtual environment
# On Windows (PowerShell):
.\venv\Scripts\Activate.ps1
# On Windows (Command Prompt):
.\venv\Scripts\activate.bat
# On Linux/macOS:
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

## Usage

Run the migration script using the virtual environment's Python:

```bash
# If activated:
python png2webp.py [path] [options]

# Or run directly via the virtual environment executable:
.\venv\Scripts\python png2webp.py [path] [options]
```

### CLI Arguments

| Argument | Shorthand | Description |
| :--- | :--- | :--- |
| `path` | | Directory containing PNG files to convert. Defaults to the current directory (`.`). |
| `--destination` | `-f` | Destination directory to save the output WebP images (retains nested subdirectory structure if `--recursive` is enabled). |
| `--quality` | `-q` | Compression quality for the WebP output from `1` to `100`. (Default: `85`). |
| `--recursive` | `-r` | Recursively scan subdirectories for PNG files. |
| `--overwrite` | `-o` | Overwrite existing WebP files with the same name. (Default: False). |
| `--delete-source` | `-d` | **Caution**: Delete the original `.png` files after successful conversion. (Default: False). |
| `--dry-run` | | Perform a trial run showing which files would be converted and how much space would be saved without writing or deleting files. |
| `--verbose` | `-v` | Enable detailed output for each file processed. |
| `--workers` | `-w` | Number of parallel worker processes to use (defaults to CPU count). |
| `--preserve` | `-p` | Preserve original file modification and access times on the output WebP images. |

### Examples

> [!TIP]
> If you have activated your virtual environment, you can run the commands using `python`. Otherwise, run them using the virtual environment's executable path directly: `.\venv\Scripts\python`.

1. **Dry-run search in the current directory** (safe preview):
   - **With venv activated**:
     ```bash
     python png2webp.py --dry-run
     ```
   - **Directly on Windows (unactivated)**:
     ```powershell
     .\venv\Scripts\python png2webp.py --dry-run
     ```

2. **Convert all PNGs in the current directory** with default settings (85% quality, lossy, does not delete original PNGs):
   - **With venv activated**:
     ```bash
     python png2webp.py -v
     ```
   - **Directly on Windows (unactivated)**:
     ```powershell
     .\venv\Scripts\python png2webp.py -v
     ```

3. **Convert a specific directory recursively, overwriting existing WebPs, and deleting original PNGs upon success**:
   - **With venv activated**:
     ```bash
     python png2webp.py C:\Users\YourUser\Pictures\ComfyUI-Outputs -r -o -d -v
     ```
   - **Directly on Windows (unactivated)**:
     ```powershell
     .\venv\Scripts\python png2webp.py "C:\Users\YourUser\Pictures\ComfyUI-Outputs" -r -o -d -v
     ```

4. **Convert images at a higher quality (90%)**:
   - **With venv activated**:
     ```bash
     python png2webp.py C:\path\to\images -q 90 -v
     ```
   - **Directly on Windows (unactivated)**:
     ```powershell
     .\venv\Scripts\python png2webp.py "C:\path\to\images" -q 90 -v
     ```

5. **Convert images and output them to a separate destination folder**:
   - **With venv activated**:
     ```bash
     python png2webp.py C:\path\to\images -f C:\path\to\destination -r -v
     ```
   - **Directly on Windows (unactivated)**:
     ```powershell
     .\venv\Scripts\python png2webp.py "C:\path\to\images" -f "C:\path\to\destination" -r -v
     ```

## Testing

A test suite `test_png2webp.py` is included to verify the image migration and metadata cleaning logic using your own test images:

1. Place your ComfyUI-generated PNG files (containing visual workflows and prompts) inside the `test_images` folder in the workspace root.
2. Run the test script:
   - **With venv activated**:
     ```bash
     python test_png2webp.py
     ```
   - **Directly on Windows (unactivated)**:
     ```powershell
     .\venv\Scripts\python test_png2webp.py
     ```
3. The script will process each PNG in `test_images`, convert it to WebP (without deleting the original PNG), and verify that:
   - The output WebP file is successfully created.
   - The `Make` EXIF tag is correctly populated with the `prompt:` JSON.
   - The `ImageDescription` EXIF tag is correctly populated with the `workflow:` JSON.
   - Large base64-encoded strings (like embedded images/masks) are successfully cleaned out of the workflow.
   - All EXIF payloads are structured and prefixed in accordance with ComfyUI specifications.

## Packaging as Standalone EXE

You can compile this Python script into a standalone Windows executable (`.exe`) that runs without requiring Python to be installed on the target machine.

### Building the EXE

1. Install PyInstaller in your virtual environment:
   ```bash
   pip install pyinstaller
   ```
2. Compile the script:
   ```bash
   pyinstaller --onefile --name png2webp png2webp.py
   ```
3. Once completed, you will find the standalone executable at:
   `dist/png2webp.exe`

### Running the EXE

You can copy `png2webp.exe` anywhere and run it directly in your shell or Command Prompt:

```powershell
# Display help and options
.\dist\png2webp.exe --help

# Run dry-run preview on current directory
.\dist\png2webp.exe --dry-run

# Run migration with verbose output on a specific folder
.\dist\png2webp.exe "C:\Users\YourUser\Pictures\ComfyUI-Outputs" -v
```
